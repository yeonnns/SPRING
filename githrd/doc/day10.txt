# day10

Log 처리

1. pom.xml에서 최신 버젼의 log4j 를 셋팅한다.

2. src/main/resources/log4j.xml 파일을 열어서 수정한다.

3. 로그처리할 클래스에서 Logger 꺼내고

4. 로그작업할 함수에서 로그 기록하고...

	log4j 
		
		Level
			FATAL	: 가장 치명적인 에러가 발생했을 때 사용
			ERROR	: 일반 에러가 발생했을 때 사용
			WARN	: 에러는 아니지만 주의할 필요가 있을 때 사용
			INFO	: 일반 정보를 나타낼 때 사용
			DEBUG	: 일반 정보를 상세하게 표현할 때 사용
			
			
		로그 메세지 패턴
			
*			%p	: priority(로그레벨) 가 출력
*			%m	: 로그 내용(메세지)
*			%d	: 로깅 이벤트가 발생한 시간
				포맷 지정
					%d{yyyy/MM/dd HH:mm:ss} 
					==> java.text.SimpleDateFormat 의 형식을 사용해서 처리
					
			%t	: 로그이벤트가 발생된 쓰레드 이름
			%%	: % 표시
*			%n	: 줄바꿈
*			%c	: 전체패키지와 클래스
			%C	: 패키지와 클래스
			%F	: 로그이벤트가 발생한 프로그램 파일 명
			%I	: 로그이벤트가 발생한 라인수
			%M	: 로그이벤트가 발생한 메소드이름
			%r	: 어플리케이션을 시작 이후부터 로그이벤트가 발생한 시간(1/1000초 단위)
